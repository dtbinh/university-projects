2 input nodes and 1 output node is correct, yes.

All nodes receive a number of inputs, which are the weighted inputs from the previous layer, or the inputs at the bottom of the network. The node then adds these inputs together, and pushes the result through a modified (continuous) step-function, to generate an output. This output is then sent on to another node or to the output, after being weighted.

Thus, each node sends an output - you can consider the node as firing if the output is close to 1, and not firing if it is close to 0, but all nodes contribute to the inputs of the next layer of the network.

Between the inputs and the first nodes, there are a set of weights. These weights can be eliminated by scaling the data if you wish, but leaving the weights is probably better as it is more flexible.

The bias term refers to the idea of attaching an input with a value of 1 to each node in the network. This allows you to effectively add a linear shift to your node's step function, by adjusting the weight between this input and the node. You do NOT have to implement the bias term in your networks, and if you do, you should include it as a flag that is OFF by default.

I hope that helps.

* You're right until step 3. You need to perform the back-propagation then after each example presented to the network. You can then store the proposed weight modifications and just update the weights with the average weight change calculated at the end of the epoch, or preferably update the weights after each training example.

* Whether you should apply sigmoids to input and output is up to you - you should consider your problem and your input and decide how you'd go forward. Typically, the inputs are scaled between 0 and 1, (or -1 and 1, depending on the activation function), though. Since you wish to predict a 0 or a 1 in binary classification, you would often like to add a activation function in your output node. If you implement a bias term in your network, adding such terms to your hidden layers and output may improve the performance a bit, but you don't need to do that to get a decent mark for the assignment.

* No, you don't need to store the output values of each node in your network for the backpropagation algorithm. We can't compare that output to any "desired" value for the node, so it won't help you. The whole idea of the backpropagation algorithm is that it takes the total error of the network, and splits it up, allocating portions to each node, according to the weights in the network. Based on these error portions, weights are updated to try reduce that apportioned error.

Hi Wolfe.

Forward propagation looks right.

As for output interpretation - your target values are 1 and 0 (1 and -1?) - while your network will output a continuous value. When you are training, the error for the forward propagation is the difference. When you later use the network to predict, you'll have to identify a threshold value (I suggest 0.5).

Your understanding of the back-propagation also sounds right to me.

dw_{ij} is the change you need to make to w_{ij}, indeed - and multiplying these weight changes first by the learning rate is exactly the right thing to do.

To determine the number of epochs, you want to look at the error rate on the test set - this is not the error of the neural network for classification, so you'll want to simply calculate the proportion of incorrect classifications using your threshold. In any case, this should decrease as the epochs increase until your network starts to overfit - then your test error will start to increase again. The point where the test error reaches is a minimum is the final network you should choose. Note that this means you should store the network weights at the end of each epoch (or at least the weights of the network with the best test error so far).

Hope this helps
